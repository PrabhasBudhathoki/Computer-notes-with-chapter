Chapter 1: Database management system

The "Database Management" chapter in Grade 12 NEB introduces students to the basics of data handling and organization. It explains the role of a Database Management System (DBMS) in storing, managing, and retrieving data efficiently, and highlights the difference between data and information.

Students also learn about relational databases, tables, and the use of SQL for managing data. The chapter covers essential commands like SELECT and INSERT, and introduces the concept of normalization to reduce data redundancy and improve consistency.

Contents

Data and Information

Data refers to raw facts and figures that have no meaning on their own. It can be in the form of numbers, text, images, audio, or video. For example, a list of student names and their marks is data. There are different types of data such as numeric data, text data, date/time data, and boolean data. Data is collected from various sources like forms, sensors, or online systems and is later processed to generate useful information. Good data should be accurate, relevant, complete, and timely. Only when data is processed and analyzed does it become meaningful and help in decision-making. Thus, data plays a key role in modern computer systems and database management.

Information is processed or organized data that has meaning and is useful for decision-making. When raw data is analyzed and presented in a structured way, it becomes information. For example, the average marks of a student calculated from individual scores is information. Information helps users understand patterns, solve problems, and make decisions. It adds context to data and turns it into something meaningful. In a database system, data is stored so it can be processed to produce useful information when needed. Good information should be accurate, timely, complete, and relevant. Without information, raw data has little value, which is why information is considered a valuable resource in both daily life and business.

Features of Information

Information has several important features that make it valuable and useful. First, it should be accurate, meaning it must be free from errors and reliable. Without accuracy, decisions made using the information can be wrong.

Information should also be timely, available when needed to support quick and effective decision-making. Completeness is another key feature — all necessary details must be included to avoid misunderstandings. Additionally, information must be relevant, meaning it should be related and useful to the specific purpose or problem.

Lastly, information should be understandable and consistent so that users can easily interpret it without confusion. These features ensure that information helps individuals and organizations make informed decisions.

Database and Its Purpose

A database is an organized collection of related data that is stored electronically for easy access and management. It helps store large amounts of information systematically so that users can retrieve and update data efficiently.

The main purpose of a database is to provide a way to store data securely and reduce redundancy, ensuring data accuracy and consistency. Databases support multiple users and applications, making it easier to manage data in businesses, schools, banks, and many other organizations. Overall, a database helps in organizing data so it can be turned into useful information quickly.

Terminologies in Database:

Table:

A table in a database is a collection of related data organized in rows and columns. Each table represents a specific type of information, such as students or products. The columns in a table are called fields, which define the different categories of data, like name, age, or price.

Field

A field is a single piece of data or a column in a database table. It represents one category or attribute of the data, such as a person’s name, age, or phone number. Each field has a specific data type and holds values for that attribute in every record of the table. Fields help organize data clearly, making it easier to search and manage information within a database.

Record

A record is a complete set of fields that represent one item or entity in the table, such as all the details of one student. In some database systems, a record is also called a tuple, which means a single row of data containing values for each field.

Tuple

A tuple is another name for a single row or record in a database table. It represents one complete set of related data values for all the fields in that table. For example, in a student table, one tuple would contain all the information about one student, such as their ID, name, and marks. Tuples help organize data into meaningful units within the database.

Object

An object is a concept from object-oriented databases, where data is stored as objects similar to real-world entities. Objects combine both data and actions (methods) that can be performed on the data.

Keys

Keys are special fields used to uniquely identify records in a table. A primary key ensures that each record is unique, while a foreign key links one table to another, helping maintain relationships between data in different tables.

Data Dictionary

A data dictionary is a collection of information about the data in a database, often called metadata. It includes details such as the names, types, sizes, and relationships of data elements. It helps in organizing, managing, and understanding the structure of the database, making it easier for users and administrators to maintain consistency and avoid redundancy.

Database Management System

Introduction

A Data Management System (DMS) is a software system that helps in storing, organizing, and managing large amounts of data efficiently. It allows users to easily create, retrieve, update, and delete data while ensuring data security, consistency, and integrity. DMS plays a vital role in handling data systematically in various fields like business, education, and healthcare.

Objectives

The main objective of a Data Management System is to store, manage, and retrieve data efficiently and securely. It aims to reduce data redundancy, ensure consistency, and provide controlled access to data for multiple users.

Advantages

A Data Management System improves data sharing and accessibility, enhances security, reduces duplication, and ensures data integrity. It also supports backup and recovery, making data management more reliable and efficient.

Disadvantages

A Data Management System improves data sharing and accessibility, enhances security, reduces duplication, and ensures data integrity. It also supports backup and recovery, making data management more reliable and efficient.

Types of Database Model

There are four types of datatypes models:

1. Hierarchical database model

In the hierarchical model, data is organized in a tree-like structure where each record has a single parent and possibly many children. This model is simple and fast for operations that follow a fixed path but is not flexible for complex relationships.

2. Network Database Model

The network model allows more complex relationships by organizing data in a graph structure. Records can have multiple parent and child records, making it more flexible than the hierarchical model but more difficult to design and manage.

3. Relational Database Model

The relational model stores data in tables (called relations) with rows and columns. It is the most widely used model today due to its simplicity, flexibility, and use of SQL for querying data. It supports easy data manipulation and avoids redundancy.

4. Object-Oriented Database Model

In this model, data is stored in the form of objects, similar to object-oriented programming. It supports complex data types and relationships, making it useful for applications that require multimedia or engineering data.

Integrity Constraints and types

Integrity constraints are rules applied to database tables to maintain the accuracy and consistency of data. They ensure that the data entered into the database follows specific rules. The main types include:

  1. Primary Key Constraint – Ensures that each record has a unique identifier.
  2. Foreign Key Constraint – Maintains referential integrity between related tables.
  3. Not Null Constraint – Ensures that a field cannot have a NULL (empty) value.
  4. Unique Constraint – Ensures all values in a column are unique.
  5. Check Constraint – Ensures that all values in a column meet a specific condition.

Normalization

a. Introduction

Normalization is a process in database design used to organize data efficiently by removing redundancy and ensuring data integrity. It involves dividing large tables into smaller, related tables and defining relationships between them. The goal is to minimize duplication and improve consistency.

b. Normal Forms

Normal forms are guidelines or rules used in the normalization process to organize database tables. Each normal form (1NF, 2NF, 3NF, etc.) addresses specific types of data anomalies and ensures higher levels of data integrity. As we move to higher normal forms, the structure becomes more refined and less redundant.

c. 1 NF, 2 NF, 3NF

First Normal form(1 NF): A table is in 1NF if all the values in each column are atomic (indivisible) and there are no repeating groups. It ensures that each column contains only one value per row.

Second Normal form(2NF): A table is in 2NF if it is in 1NF and all non-key attributes are fully functionally dependent on the entire primary key (no partial dependency). It applies mostly to tables with composite primary keys.

Third Normal form(3 NF): A table is in 3NF if it is in 2NF and all non-key attributes are not transitively dependent on the primary key. This means non-key fields depend only on the key, not on other non-key fields.

d. Advantages:

  • Reduces data redundancy (no repeating data).
  • Ensures data integrity and consistency.
  • Makes updating data easier and more accurate.
  • Saves storage space by avoiding duplicate data.
  • Improves database organization and understanding.

e. Disadvantages:

  • Increases the number of tables, making queries more complex.
  • Requires more joins, which can reduce performance.
  • Slower data retrieval in some cases.
  • Difficult to design and manage for beginners.
  • Not always suitable for reporting and analytics.

Centralized and Distributed database

1. Centralized Database

A centralized database is stored in a single central server or location, and all users access data through a network. It is easy to manage, maintain, and secure since everything is in one place. However, it creates a dependency on one system — if the central server fails, the entire database becomes unavailable, and access speed may be slower for remote users.

Advantages:

  • Easy to manage and maintain.
  • Simple backup and recovery.
  • Strong data security in one location.
  • Consistent and accurate data.
  • Less hardware and software cost.

Disadvantages:

  • Single point of failure.
  • Heavy load on one server.
  • Slower access for remote users.
  • Network dependency.
  • Limited scalability.

2. Distributed Database

A distributed database is spread across multiple servers or locations but functions as a single system for users. Each site stores part of the data and may process queries locally. This increases reliability, fault tolerance, and speed for local users. However, it is more complex to manage, requires synchronization, and can have higher maintenance costs.

Advantages:

  • Faster access for local users.
  • High reliability and availability.
  • Continues working even if one site fails.
  • Better scalability.
  • Load is distributed across servers.

Disadvantages

  • Complex system design and management.
  • Expensive setup and maintenance.
  • Difficult data synchronization.
  • Security management is harder.
  • Complex backup and recovery.

Database Security

Introduction

Database security refers to the measures used to protect a database from unauthorized access, misuse, or theft. It ensures that only authorized users can access or modify data, keeping sensitive information safe. Security includes user authentication, access control, encryption, and regular backups to prevent data loss or breaches.

Challenges

Database security faces several challenges like unauthorized access, SQL injection attacks, and insider threats. Ensuring data privacy while allowing access to multiple users is complex. Managing strong authentication, encryption, and regular updates requires effort and expertise. Also, backup and disaster recovery need to be handled carefully to avoid data loss.

Security measures

To protect data from unauthorized access and misuse, several security measures are implemented in database systems. User authentication using usernames and passwords ensures only authorized users can log in. Access control assigns specific permissions to different users, restricting who can read, write, or delete data. Data encryption is used to protect sensitive information during storage and transmission. Firewalls and intrusion detection systems help monitor and block suspicious activities. Regular backups and update patches are also essential to recover from system failures or security breaches. Together, these measures help maintain data confidentiality, integrity, and availability.

DBA and roles of DBA

Database Administrator (DBA)

A Database Administrator (DBA) is a person responsible for the management, maintenance, and security of a database system. In today's world, data is considered one of the most valuable assets of an organization, and databases help store, organize, and manage this data efficiently.

Roles of DBA

  1. Database Installation and Configuration: The DBA is responsible for installing and setting up the database software on servers and ensuring it works correctly.
  2. Backup and Recovery: DBAs regularly back up data and create recovery strategies to prevent data loss during failures or disasters.
  3. Performance Tuning: DBAs monitor database performance and make necessary adjustments to ensure fast and efficient data access.
  4. Security Management: They control user access to the database, ensuring that only authorized users can access or modify data.
  5. Data Integrity and Maintenance: The DBA ensures the consistency, accuracy, and reliability of data within the database.

ER Diagrams

An ER Diagram is a visual representation of entities and the relationships between them in a database. It is used during the database design process to understand how data is connected and structured. In an ER diagram, entities are objects or things (like students or books) that can be identified distinctly. Attributes describe properties of an entity (e.g., a student has a name, ID, and address).

Symbols and their meanings

  1. Rectangles for entities
  2. Ellipses for attributes
  3. Diamonds for relationships
  4. Lines connecting entities to attributes and relationships
  5. Double ellipses for multivalued attributes
  6. Underlined attributes to represent primary keys

SQl: DDl and DML

SQL is a language used to communicate with databases. It is mainly divided into two categories: DDL (Data Definition Language) and DML (Data Manipulation Language).

DDL includes commands like:

  • CREATE (to create new tables or databases),
  • ALTER (to modify existing structures),
  • DROP (to delete tables or databases).

DML includes commands likeL:

  • INSERT (to add new data),
  • UPDATE (to modify existing data),
  • DELETE (to remove data),
  • SELECT (to retrieve data from one or more tables).

Data constraints

Data constraints are rules applied to columns in a table to ensure data accuracy and integrity. Some commonly used constraints are:

  • NOT NULL: Ensures that a column cannot have a NULL (empty) value.
  • UNIQUE: Ensures all values in a column are different.
  • PRIMARY KEY: A combination of NOT NULL and UNIQUE. It uniquely identifies each record in a table.
  • AUTO INCREMENT: Automatically increases the value of a numeric field (usually for primary keys) whenever a new record is inserted.

Operators: AND, OR, NOT

Operators in SQL are used to filter data based on specific conditions.

  • AND: Returns rows when all conditions are true (e.g., WHERE age > 18 AND gender = 'Male').
  • OR: Returns rows when at least one condition is true (e.g., WHERE city = 'Kathmandu' OR city = 'Lalitpur').
  • NOT: Reverses the result of a condition (e.g., WHERE NOT (status = 'Inactive')).

WHERE clause

The WHERE clause is used to filter records in a table based on a given condition. It helps retrieve specific data rather than the entire 

For example:

                SELECT * FROM students WHERE grade > 60;
This command retrieves only those students who scored more than 60.

ORDER BY clause

The ORDER BY clause is used to sort the result set of a query in either ascending (ASC) or descending (DESC) order. 

For example:

                SELECT * FROM students ORDER BY name ASC;
This command displays all student records sorted by name in alphabetical order.

SQl Joins

SQL JOINs are used to combine rows from two or more tables based on a related column.

1. INNER JOIN: Returns only the records that have matching values in both tables. 

    SELECT students.name, books.title
    FROM students
    INNER JOIN books ON students.id = books.student_id;

2. LEFT JOIN (or LEFT OUTER JOIN): Returns all records from the left table and the matched records from the right table. Unmatched records from the right table will have NULL values. 

    SELECT students.name, books.title
    FROM students
    LEFT JOIN books ON students.id = books.student_id;

3. RIGHT JOIN (or RIGHT OUTER JOIN): Returns all records from the right table and the matched records from the left table. Unmatched records from the left table will have NULL values. 

    SELECT students.name, books.title
    FROM students
    RIGHT JOIN books ON students.id = books.student_id;

4. FULL JOIN (or FULL OUTER JOIN): Returns all records when there is a match in either left or right table. If no match is found, NULL is returned for the missing side. 

    SELECT students.name, books.title
    FROM students
    FULL OUTER JOIN books ON students.id = books.student_id;
